# Managing Resources

This guide explains how to add and manage downloadable resources in the Afrexia CMS.

## Overview

Resources are downloadable documents that provide additional value to website visitors. They include product catalogs, technical specifications, certificates, compliance documents, and guides.

## Resource Structure

A resource document contains:

- **Basic Information**: Title, description
- **File**: The downloadable document
- **Organization**: Category, language
- **Relationships**: Related products
- **Publishing**: Date and display order
- **Workflow**: Draft/Review/Published status

## Creating a New Resource

### Step 1: Access Resources

1. Click **"Resource"** in the sidebar
2. Click **"+ Create"** button

### Step 2: Basic Information

#### Title (Required)

Enter the resource title in both languages:
- **French**: e.g., "Catalogue Produits 2024"
- **English**: e.g., "Product Catalog 2024"

**Tips**:
- Be descriptive and specific
- Include year for dated documents
- Use professional terminology
- Keep it concise

#### Slug (Required)

Generate URL-friendly slug:
1. Click **"Generate"** button

**Example**: `product-catalog-2024`

**Tips**:
- Slug is auto-generated from English title
- Edit manually if needed
- Use lowercase and hyphens only

#### Description (Optional but Recommended)

Write a brief description in both languages:

**French**:
```
Catalogue complet de tous nos produits avec spécifications 
techniques et conditions commerciales.
```

**English**:
```
Complete catalog of all our products with technical 
specifications and commercial terms.
```

**Tips**:
- Explain what the document contains
- Mention key information included
- Keep it 1-2 sentences
- Helps users decide if they need it

### Step 3: Category (Required)

Select the resource category:

- **Product Catalogs**: General product listings
- **Technical Specifications**: Detailed product specs
- **Certifications**: Official certificates
- **Compliance Documents**: Regulatory documentation
- **Guides & Manuals**: Educational materials
- **Case Studies**: Success stories and examples
- **Other**: Miscellaneous documents

**Tips**:
- Choose the most appropriate category
- Categories help organize the Resources page
- Users can filter by category

### Step 4: File Upload (Required)

Upload the document:

1. Click **"Upload"** button
2. Select file from your computer
3. Wait for upload to complete

**Supported Formats**:
- PDF (recommended)
- DOC, DOCX (Microsoft Word)
- XLS, XLSX (Microsoft Excel)

**File Guidelines**:
- **Max size**: 10MB
- **Recommended**: Under 5MB for faster downloads
- **Format**: PDF is preferred for universal compatibility
- **Quality**: Ensure text is readable and images are clear

**Tips**:
- Compress large PDFs before uploading
- Use descriptive filenames
- Ensure documents are finalized (no draft watermarks)
- Test download after uploading

### Step 5: Language (Required)

Select the document language:

- **French (fr)**: Document is in French only
- **English (en)**: Document is in English only
- **Both**: Document contains both languages

**Tips**:
- Choose "Both" for bilingual documents
- Create separate resources for single-language versions
- Clearly indicate language in title if single-language

### Step 6: Related Products (Optional)

Link to related products:

1. Click **"Select"**
2. Search for product name
3. Click to select
4. Repeat for multiple products

**When to use**:
- Technical specifications for specific products
- Product-specific certificates
- Case studies featuring products

**Tips**:
- Link relevant products only
- Resources appear on linked product pages
- Helps buyers find product-specific documentation

### Step 7: Publishing Details

#### Published At (Optional)

Set the publication date:
1. Click the date field
2. Select date from calendar

**Tips**:
- Use document creation or update date
- Helps users see if information is current
- Leave blank if not applicable

#### Display Order (Optional)

Set the display order (number):
- Example: 1, 2, 3, etc.

**Tips**:
- Lower numbers appear first
- Use to prioritize important documents
- Group related documents with similar numbers
- Leave blank for default alphabetical order

### Step 8: Workflow Status

Set the document status:
- **Draft**: Work in progress
- **In Review**: Ready for approval
- **Published**: Available for download

### Step 9: Publish

1. Review all fields
2. Test download the file
3. Click **"Publish"** button
4. Resource appears on website immediately

## Editing Existing Resources

### Finding a Resource

1. Click **"Resource"** in sidebar
2. Use search bar to find by title
3. Or filter by category
4. Click the resource to open

### Updating a File

To replace a document:
1. Open the resource
2. Click **"Upload"** in File field
3. Select new file
4. Old file is replaced
5. Click **"Publish"** to update

**When to update**:
- Annual updates (catalogs, certificates)
- Corrections or revisions
- Expired certificates replaced with new ones
- Updated compliance documents

### Archiving Old Resources

To remove outdated resources:
1. Open the resource
2. Click **"•••"** menu (top right)
3. Select **"Unpublish"**
4. Or delete if no longer needed

## Resource Checklist

- [ ] Title in French and English

## Best Practices

### Document Quality

- **Professional formatting**: Use consistent styles and branding
- **Clear content**: Ensure text is readable and well-organized
- **Updated information**: Keep documents current
- **Proper naming**: Use descriptive filenames
- **Optimized size**: Compress large files

### Organization

- **Logical categories**: Place documents in appropriate categories
- **Consistent naming**: Use clear, descriptive titles
- **Version control**: Include version numbers or dates in titles
- **Regular review**: Audit resources quarterly

### Accessibility

- **PDF/A format**: Use for long-term archival
- **Searchable text**: Ensure PDFs have selectable text (not scanned images)
- **Bookmarks**: Add bookmarks to long documents
- **Alt text**: Include for images in documents
- **Clear structure**: Use headings and lists

### Security

- **Remove sensitive data**: Don't include confidential information
- **Watermarks**: Consider adding watermarks to proprietary documents
- **Version control**: Track document versions
- **Access control**: Use workflow status to control visibility

## Common Resource Types

### Product Catalog

**Contents**:
- Product listings with photos
- Brief descriptions
- Key specifications
- Pricing (if applicable)
- Contact information

**Format**: PDF, 10-30 pages
**Update frequency**: Annually or seasonally

### Technical Specification Sheet

**Contents**:
- Detailed product specifications
- Quality metrics
- Packaging information
- Certifications
- HS codes and Incoterms

**Format**: PDF, 1-4 pages per product
**Update frequency**: As specifications change

### Certificate

**Contents**:
- Official certification document
- Issuing body information
- Validity dates
- Scope of certification

**Format**: PDF, 1-2 pages
**Update frequency**: When renewed

### Compliance Guide

**Contents**:
- Regulatory requirements
- Compliance procedures
- Documentation requirements
- Contact information

**Format**: PDF, 5-20 pages
**Update frequency**: When regulations change

### Incoterms Guide

**Contents**:
- Explanation of each Incoterm
- Responsibilities breakdown
- Cost allocation
- Risk transfer points

**Format**: PDF, 5-10 pages
**Update frequency**: When Incoterms update (every 10 years)

## Common Issues

### File Won't Upload

**Possible causes**:
- File too large (over 10MB)
- Unsupported format
- Network issue

**Solution**:
- Compress the file
- Convert to PDF
- Try again with stable connection

### Resource Not Appearing on Website

**Possible causes**:
- Workflow status not "Published"
- Missing required fields

**Solution**:
- Check workflow status
- Verify all required fields are complete

### Download Link Not Working

**Possible cause**:
- File upload incomplete

**Solution**:
- Re-upload the file
- Verify file appears in CMS
- Test download from CMS

### Wrong File Downloaded

**Possible cause**:
- File was replaced but cache not cleared

**Solution**:
- Wait a few minutes for cache to clear
- Clear browser cache
- Verify correct file in CMS

## Resource Maintenance

### Regular Tasks

**Monthly**:
- Review download statistics
- Check for broken links
- Update popular resources

**Quarterly**:
- Audit all resources
- Remove outdated documents
- Update certificates and compliance docs

**Annually**:
- Update product catalogs
- Refresh all guides and manuals
- Review and update case studies

### Version Control

For documents that change frequently:

**Naming convention**:
- Include version number: "Product Catalog v2.0"
- Or include date: "Product Catalog 2024"

**Change log**:
- Maintain a change log for major documents
- Note what changed in each version
- Include in document or description
