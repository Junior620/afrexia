# CMS Best Practices

This guide provides best practices for managing content in the Afrexia CMS effectively and efficiently.

## Content Strategy

### Planning

**Develop a content calendar**:
- Plan content 1-3 months ahead
- Align with business goals
- Consider seasonal topics
- Balance content types

**Set clear objectives**:
- Define purpose for each piece
- Identify target audience
- Establish success metrics
- Plan distribution strategy

**Research thoroughly**:
- Understand your audience
- Research keywords
- Analyze competitors
- Stay current with industry trends

### Content Types Balance

**Maintain variety**:
- 40% Educational content (guides, how-tos)
- 30% Industry insights (market analysis, trends)
- 20% Company content (products, certifications)
- 10% Promotional content (case studies, success stories)

## Writing Best Practices

### Tone and Voice

**Professional but accessible**:
- Use industry terminology correctly
- Explain complex concepts clearly
- Avoid jargon when possible
- Maintain B2B professional tone

**Consistent voice**:
- Establish brand voice guidelines
- Train all content creators
- Review for consistency
- Update guidelines as needed

### Structure and Formatting

**Use clear hierarchy**:
```
H1: Page title (automatic)
H2: Main sections
H3: Subsections
H4: Minor subsections
```

**Break up content**:
- Short paragraphs (2-4 sentences)
- Bulleted lists for easy scanning
- Numbered lists for steps
- Subheadings every 200-300 words

**Emphasize key points**:
- **Bold** for important terms
- *Italic* for emphasis
- > Blockquotes for citations
- Links for references

### SEO Writing

**Keyword usage**:
- Include primary keyword in title
- Use keyword in first paragraph
- Sprinkle naturally throughout
- Don't stuff keywords

**Optimize for search**:
- Write compelling meta descriptions
- Use descriptive headings
- Include internal links
- Add external authoritative links
- Optimize images with alt text

**Write for humans first**:
- Focus on value and readability
- Natural language over keyword density
- Answer user questions
- Provide actionable insights

## Image Best Practices

### Image Quality

**Technical requirements**:
- Minimum 1200px width
- High resolution (72-150 DPI)
- Proper format (JPG for photos, PNG for graphics)
- Optimized file size (under 2MB)

**Visual quality**:
- Professional photography
- Good lighting and composition
- Sharp focus
- Relevant to content

### Image Optimization

**Before uploading**:
- Resize to appropriate dimensions
- Compress without quality loss
- Convert to web-friendly format
- Remove metadata if needed

**Tools for optimization**:
- TinyPNG (https://tinypng.com)
- ImageOptim (Mac)
- Squoosh (https://squoosh.app)
- Photoshop "Save for Web"

### Alt Text

**Write descriptive alt text**:
- Describe what's in the image
- Keep it concise (125 characters max)
- Include keywords naturally
- Don't start with "Image of..."

**Examples**:
- ❌ "Image of cocoa"
- ✅ "Premium cocoa beans drying in the sun"
- ❌ "Picture"
- ✅ "Farmer inspecting coffee cherries on plantation"

## Multilingual Content

### Translation Best Practices

**Maintain consistency**:
- Use consistent terminology
- Keep meaning equivalent
- Adapt culturally when needed
- Review both versions together

**Don't use machine translation**:
- Use professional translators
- Or native speakers
- Review and edit carefully
- Maintain quality standards

**Consider cultural differences**:
- Adapt examples and references
- Use appropriate units (metric/imperial)
- Consider local regulations
- Respect cultural sensitivities

### Translation Workflow

1. **Write in primary language** (usually English)
2. **Review and finalize** primary version
3. **Translate to second language** (French)
4. **Review translation** for accuracy
5. **Publish both versions** simultaneously

## File Management

### Document Organization

**Naming conventions**:
- Use descriptive names
- Include version or date
- Use lowercase and hyphens
- Be consistent

**Examples**:
- `product-catalog-2024-v2.pdf`
- `cocoa-technical-spec-jan2024.pdf`
- `iso-22000-certificate-2023.pdf`

### File Optimization

**PDFs**:
- Compress before uploading
- Ensure text is searchable
- Add bookmarks for long documents
- Use PDF/A for archival

**Office documents**:
- Convert to PDF when possible
- Remove comments and tracked changes
- Embed fonts
- Optimize images within document

### Version Control

**Track versions**:
- Include version number in filename
- Maintain change log
- Archive old versions
- Document major changes

**Update process**:
1. Create new version
2. Upload to CMS
3. Update resource document
4. Archive old version
5. Notify users if needed

## Common Pitfalls to Avoid

### Content Issues

**Don't**:
- ❌ Copy content from other sources
- ❌ Use machine translation without review
- ❌ Publish without proofreading
- ❌ Ignore SEO best practices
- ❌ Forget to fill both languages
- ❌ Use low-quality images
- ❌ Skip the review process

**Do**:
- ✅ Create original, valuable content
- ✅ Use professional translation
- ✅ Proofread thoroughly
- ✅ Optimize for search engines
- ✅ Complete all language versions
- ✅ Use high-quality images
- ✅ Follow workflow process

### Technical Issues

**Don't**:
- ❌ Upload huge files
- ❌ Use unsupported formats
- ❌ Forget alt text
- ❌ Leave broken links
- ❌ Skip testing
- ❌ Ignore mobile users

**Do**:
- ✅ Optimize file sizes
- ✅ Use recommended formats
- ✅ Add descriptive alt text
- ✅ Test all links
- ✅ Preview before publishing
- ✅ Check mobile display
